## The NOTICE Invariant: Polymorphic License Carrier

This project employs a *layered licensing approach* as declared in the frontmatter and/or LICENSE. To ensure these layers survive distribution, compilation, and the ontological transition at 1.x.y, the following stipulation is in order:

### Canonical Anchor

The `build_files/NOTICE` file is hereby designated as the polymorphic license invariant the single source of truth for all rights, obligations, and epistemological claims attached to any artifact produced by or derived from this project. The Apache License originally stipulated this clause; this project extends and utilizes it.

### Polymorphism Defined

The NOTICE file transitions its role based on context and artifact type:

| Context | Role of NOTICE |
|---------|----------------|
| Source repository (pre-1.x.y) | Informational attribution (Apache-style notice) |
| Source repository (post-1.x.y) | The canonical license reference; supersedes any root-level LICENSE |
| Compiled/derivative artifact | The sole license the terms governing use, modification, and redistribution are embedded within the artifact itself |
| Quineic derivative (CC0-bound) | The NOTICE is the watermark; its presence is the only proof of origin |

### Relationship to the Layered Licensing Regime

The NOTICE carries all three licensing layers forward into any derivative:

1. _BSD 3-Clause_: applies to individual source files
2. _CC BY-NC-SA 4.0_: applies to distributed packages (source, binaries, accompanying materials)
3. _CC0 1.0_: applies to Quineic derivatives (auto-generated code/data)

The NOTICE is the mechanism by which these layers persist. Without the NOTICE, the layers collapse and the artifact is considered outside the scope of this project's epistemological guarantees.

### Bare-Minimum Requirement

Any distribution, fork, compilation output, or derivative _must_ include the `build_files/NOTICE` file verbatim and intact. Failure to do so:

- Voids any claim to the NP-hard reversal guarantee (see below)

### Architectural Invariant

This invariant is testable and enforceable via CI checks that verify:

- The NP-hard reversal guarantee applies only to the intentional pipeline from known source through known toolchain(s) to known derivative(s).

This is a feature. It means the project's protections are bounded by the same epistemological limits as all macroscopic systems. We do not pretend to transcend physics.
